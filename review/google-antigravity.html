What is Google Antigravity?

Google Antigravity is Google's agent-first development platform. The 2.0 release, announced at Google I/O on May 19, 2026, is not just a code editor — it is a five-surface platform: a desktop app, a CLI called agy, an SDK for building custom agents, a Managed Agents API delivered through the Gemini API, and an Enterprise Agent Platform. The platform is optimized for Gemini 3.5 Flash, which Google says was co-developed using Antigravity itself, with multi-model support for Anthropic's Claude Sonnet 4.5 and GPT-OSS.

The original 1.0 launched in November 2025 as an AI-powered IDE competing with Cursor and Windsurf. The 2.0 release reframes the product around multi-agent orchestration — running several specialized subagents in parallel across distinct coding tasks rather than driving a single Composer-style agent through one task at a time. That's the headline capability, and on paper it's a real category shift.

This review is based on Google's documented features and pricing, primary-source coverage of the I/O announcement, and community sentiment across developer forums, TechCrunch, The Next Web, and independent reviews — not hands-on testing.

What changed

Version 2.0 launched May 19, 2026 at Google I/O. The product expanded from a single IDE to a five-surface platform with a new agy CLI, an SDK, and a Managed Agents API. A new $100/month AI Ultra tier sits between the existing Pro subscription and the $200/month top tier, which dropped from a former $250 price point. The Gemini CLI is being retired June 18, 2026 — every Gemini CLI user must migrate to the Antigravity CLI before that date. The 2.0 update was pushed automatically on launch day, wiping stored configurations and removing the built-in code editor from existing user installations.

The Five-Surface Platform

Antigravity 2.0 is structured around five distinct surfaces rather than a single application. Understanding what each one does is the difference between treating Antigravity as "Google's Cursor" and treating it as the agent platform Google clearly wants it to be.

  • Desktop app — the original IDE surface. Code editor, chat panel, and direct access to subagents from the editor.
  • agy CLI — a Go-built command-line interface that replaces the Gemini CLI. Intended as the terminal-first entry point for agents, scripting, and pipeline integration.
  • SDK — for building and deploying custom agents that fit your team's workflow rather than relying on Google's defaults.
  • Managed Agents API — agents-as-a-service via the Gemini API, for embedding agent behavior into third-party products without standing up your own infrastructure.
  • Enterprise Agent Platform — admin, billing, audit, and policy controls for organizations deploying agents across teams.

The Manager Surface inside the desktop app is the new central hub — an asynchronous dashboard for orchestrating and monitoring multiple agents at once. Dynamic subagents can be spun up on demand for split workflows, and scheduled background tasks let agents run automatically while you work on something else. That's the differentiated capability story in one paragraph.

Multi-Agent Orchestration — The Headline Feature

Parallel multi-agent execution is where Antigravity separates itself from the rest of the category. Cursor runs a single Composer agent through one task at a time. Windsurf — now rebranded Devin Desktop — has a comparable single-agent core, though its new Agent Command Center is beginning to track multiple agents. Antigravity is built around the assumption that you'll dispatch several specialized subagents simultaneously across distinct tasks — one refactoring a module, another writing tests, another updating documentation, another running a scheduled lint pass — and review the outputs from a Manager Surface.

If multi-agent workflows are the future of coding assistants, this is the most committed bet on that thesis from any major vendor in 2026. The honest question is whether the orchestration UX, agent reliability, and cost economics support that bet under sustained daily use. That's what 60–90 days of usage in the wild will tell us.

Google eating its own cooking

Google says Gemini 3.5 Flash was itself co-developed using Antigravity. That's a credibility signal worth noting — using your own platform to build the underlying model is the strongest possible internal validation. It does not, however, fix the launch execution problems below.

The Launch Failure — Read This Before You Migrate

This is the most important thing for developers to know about Antigravity in May 2026, and there's no soft way to put it. The 2.0 launch was a trust failure, and it happened in three specific ways that anyone considering this platform should weigh before committing.

First: the auto-update wiped configurations. Google pushed the 2.0 update automatically on launch day. For existing Antigravity 1.0 users, that meant stored configurations were wiped, and the built-in code editor was removed in favor of the new platform structure. There was no opt-in, no migration window, and no clear path to restore prior state. Developers logged in to a setup that no longer matched the one they'd been using the day before.

Second: the replacement CLI wasn't ready. Google announced at I/O that the Gemini CLI is being retired on June 18, 2026 — a 27-day window — and that all users must migrate to agy, the new Antigravity CLI. At launch, agy was not available on Homebrew, apt, npm, or any other public package manager. The replacement for the tool being mandatorily retired was not installable. Telling developers "your current tool ends in 27 days; the new one isn't shippable yet" is the kind of operational miss that defines how a platform launch is remembered.

Third: the open-source-to-closed-source change. Gemini CLI was open source. Antigravity CLI is closed source. For developers and teams that chose Gemini CLI specifically because they could read, audit, fork, and self-host the code, the forced migration removes that property. It's a meaningful change for the security-conscious and the regulated, and it was not framed as such in the I/O announcement.

Each of those decisions on its own would be a papercut. Together they are a signal about how the team treats developers who already depend on the product. That signal matters more than any individual feature.

Who Should Use Antigravity?

Antigravity is a strong evaluation candidate for two specific audiences. The first is Google ecosystem developers — anyone building on Android, Firebase, or Google Cloud, where Antigravity's native integrations remove the most friction. The second is teams that want parallel multi-agent execution at a lower per-month cost than Cursor Ultra or Claude Max, with the willingness to absorb a noisy launch period.

Who should look elsewhere

If you need a stable production coding tool in May 2026, this is not it yet. The platform deserves an evaluation, not a commitment. Developers outside the Google ecosystem will get less marginal value than the marketing implies, since the Android/Firebase integrations are where the differentiation is densest. And anyone who chose Gemini CLI specifically for its open-source license has a real reason to push back against the forced closed-source migration — that's not a workflow preference, it's a procurement decision. Cursor remains the most capable IDE-first option, Windsurf the most beginner-friendly agentic editor, and GitHub Copilot the default for teams already on GitHub.

AIToolGrade Take

Antigravity 2.0 is the most technically ambitious agentic coding platform launched in 2026 — and the most poorly executed launch in the category. The product vision is clear and differentiated: parallel multi-agent orchestration at lower cost than Cursor or Claude Code, with the deepest Google ecosystem integration available. The execution on launch day was a trust failure. Wiping configurations automatically, shipping without an installable CLI, and mandating a 27-day migration with no working replacement are not minor papercuts — they are decisions that tell developers how the company treats production workflows. Our recommendation: evaluate Antigravity 2.0 seriously, but wait 60–90 days for the CLI to ship properly and documentation to catch up before committing it to production work. The underlying platform deserves a fair evaluation when it works.

The Bottom Line

Antigravity 2.0 is a real bet on a different shape of AI coding — multiple specialized agents working in parallel rather than a single agent driven through tasks one at a time. If that thesis plays out, this is the platform best positioned to lead it, especially for teams already on Android, Firebase, and Google Cloud. The five-surface architecture, the $100 Ultra tier, and the multi-model support inside one environment are the right ingredients.

The case against adopting it today is narrow but firm. The launch wiped configs and shipped without an installable CLI. The 27-day forced migration to a closed-source replacement is a credibility issue, not a minor papercut. Documentation hasn't caught up. None of these are fatal — they are recoverable inside a quarter — but they are the reason this scores 7.5 and not higher. Evaluate seriously, hold off on production migration, and revisit in Q3 2026 when the CLI and the docs have had a chance to settle.
